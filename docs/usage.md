## 快速开始

### 首次访问

1. 打开浏览器访问 `http://<主机IP>:3000`
2. 首次访问会跳转到 `/login`，设置一个访问口令（最少 4 位）
3. 设置成功后自动登录

### 配置 AI 接口

1. 点击顶部导航栏的 ⚙ 设置按钮
2. 在 AI 连接区域填写：
   - **接口密钥**：你的 API Key
   - **接口基础地址**：如果使用第三方代理/中转，修改此地址
   - **默认模型**：点击刷新按钮获取可用模型列表，或手动输入模型名
3. 测试连接是否正常：模型列表成功加载即为正常

### 配置媒体库

1. 点击顶部导航栏"媒体库"
2. 添加一个或多个容器内路径（注意是**容器内路径**，不是宿主机路径）
3. 例如 Docker 挂载了 `/volume1/video:/media/movies`，则应填写 `/media/movies`
4. 点击"批量检测所有路径"确认挂载和权限正常
5. 保存配置

### 开始第一个翻译任务

1. 返回首页，在文件浏览器中选择一个视频文件（支持 `.mkv`、`.mp4`、`.avi`、`.webm`、`.ts`）
2. 左侧视频窗格会自动探测字幕轨道，选择需要的轨道
3. 右侧翻译选项区域设置参数
4. 点击"开始翻译"

## 设置页面

### AI 连接

| 设置项 | 说明 |
|--------|------|
| 接口密钥 | 大语言模型的 API Key，必填 |
| 接口基础地址 | 默认 `https://api.openai.com/v1`，兼容 OpenAI 协议的第三方服务需修改 |
| 默认模型 | 用于翻译的模型名称，支持获取列表或手动输入 |

### 默认翻译行为

| 设置项 | 说明 |
|--------|------|
| 目标语言 | 翻译的目标语言，默认 `zh-CN`（简体中文） |
| 输出模式 | 仅译文 / 双语对照 / 仅导出原字幕 |
| 翻译模式 | 非流式（推荐）/ 流式（兼容性较弱） |

### 性能参数

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 分块大小 | 5000 | 估算的 token 数上限，越大单次 AI 调用处理的条目越多 |
| 并发任务数 | 3 | 同时运行的任务数量上限 |
| 最大重试次数 | 2 | 单块翻译失败后的重试次数 |
| 日志保留天数 | 5 | 任务日志和 AI 日志的保留天数 |

> **分块大小建议**：模型上下文越大（如 128K），此值可以设得越高。字幕条目较少（<200 条）时设 8000-10000 可使所有条目合并到一个块中一次翻译完成。

### 高级设置

| 设置项 | 说明 |
|--------|------|
| 流式使用量统计 | 仅在使用流式模式时建议开启，可获取精确 token 用量 |
| 术语表 | 自定义翻译术语，格式为键值对，如 `Jedi:绝地` |

## 文件浏览与字幕选择

### 文件浏览器布局

页面分为左右两个可调整大小的窗格：

- **左侧**：目录树，支持展开/折叠，点击目录名旁的箭头展开子目录
- **右侧**：选中的视频/字幕文件详情

### 选择视频文件

点击 `.mkv`、`.mp4`、`.avi` 等视频文件后，系统会自动探测内嵌字幕轨道，在右侧显示：

- 轨道编号、语言、编码等信息
- 绿色标记表示支持的字幕轨道
- 点击"选择此轨道"确认

### 直接使用外挂字幕

直接点击 `.srt`、`.vtt`、`.ass`、`.ssa` 文件（无需先选视频），系统将以此外挂字幕为源进行翻译。

### 翻译选项

开始翻译前可在右侧面板调整：

| 选项 | 说明 |
|------|------|
| 输出模式 | 仅译文 / 双语 / 仅原文字幕 |
| 字幕格式 | SRT / ASS / SRT+ASS |
| 字幕样式 | 仅 ASS 格式生效，包括简洁双语、影院风格、学习模式、紧凑、单色 |
| 双语排版 | 译文在上 / 原文在上 |
| 翻译风格 | 可选用预设风格（默认/犯罪/间谍/古装/喜剧/科幻） |

## 翻译任务管理

### 任务状态

| 状态 | 说明 |
|------|------|
| queued | 已创建，等待执行 |
| extracting | 正在从视频提取字幕轨道 |
| parsing | 正在解析字幕文件并分块 |
| translating | 正在进行 AI 翻译 |
| exporting | 正在合成导出文件 |
| done | 翻译完成 |
| error | 翻译失败 |
| cancelled | 用户手动取消 |
| review | 待核对（已废弃，现在所有任务完成即 done） |

### 查看任务进度

在 `翻译历史` 中点击任务行的 👁 按钮，进入任务详情页：

- 实时进度百分比
- 分步日志（提取 → 分块 → 翻译 → 导出）
- 每个块的校验日志，包括"未改动原文"统计
- 诊断信息：连接状态、翻译模式、token 用量

### 取消任务

在任务详情页点击"取消任务"按钮。取消后：

- 已发起的 API 调用会被立即中断（通过 AbortController）
- 已翻译完成的部分保留
- 任务状态变为 `cancelled`

### 重新翻译

在 `翻译历史` 中点击 🔄 按钮。系统会创建一个新任务：

- 跳过本地缓存，强制重新翻译
- 使用与原任务相同的设置

### 下载字幕

- 点击 ⬇ 按钮下载翻译结果
- 任务格式选择 `both` 时，会同时显示 SRT 和 ASS 下载按钮
- 任务详情页还有"下载原始"按钮，下载视频原始内嵌字幕

### 清空历史

点击"清空历史"按钮，可删除所有已完成/失败/已取消/待核对的任务，同时清理：

- `task_logs` 表
- `task_review_entries` 表
- `task_responses` 表
- `translations_cache` 表
- `db/logs/` 目录中的任务日志文件

## 核对页面

### 进入方式

所有**已终止的任务**（done / error / cancelled / review）在翻译历史页都会显示 📋 按钮，点击进入核对页。

### 页面布局

核对页分为左右两栏：

- **左栏**：字幕条目列表，支持筛选、选择、编辑
- **右栏**：实时预览面板

### 筛选功能

顶部筛选区域：

| 筛选器 | 选项 | 说明 |
|--------|------|------|
| 状态筛选 | 全部 / 待核对 / 已编辑 / 仅已选 | 按条目状态过滤 |
| 原因筛选 | 全部原因 / 缺少译文 / 译文与原文相同 / ... | 按 AI 校验原因过滤 |

### 快捷选择按钮

- **全选可见**：选中当前筛选结果中的所有条目
- **待核对**：选中所有状态为 `needs_review`、`fallback_original`、`missing` 的条目
- **清空选择**：取消所有选择

### 逐条操作

每个字幕条目包含三栏：

| 栏 | 内容 | 操作 |
|----|------|------|
| 原文 | 原始字幕文本 | 不可编辑 |
| 当前译文 | AI 翻译结果 | 可点击 ↑↓ 将译文与相邻条目互换 |
| 最终导出 | 即将写入文件的文本 | 可编辑 (Textarea)、恢复译文、使用原文、↑↓ 互换 |

### 批量操作

| 按钮 | 功能 |
|------|------|
| 批量重译 | 对选中的条目重新发起 AI 翻译 |
| 设为原文 | 将选中条目的最终文本设为原文 |
| 所选上移/下移 | 整体移动选中的条目顺序 |
| 译文下移/上移 | 从第一个选中条目起，所有译文字段整体偏移一行（用于修正 AI 错位） |
| 保存修改 | 将当前所有编辑提交到数据库 |

### 实时预览

右侧预览面板支持：

- **格式切换**：SRT / ASS
- **双语排版**：译文在上 / 原文在上
- **自动刷新**：切换导出模式、字幕样式时自动刷新
- **手动刷新**：点击"刷新"按钮

### 导出最终字幕

1. 在核对页完成所有编辑和确认
2. 在顶部选择导出设置：
   - **导出模式**：仅译文 / 双语 / 仅原文
   - **字幕样式**：5 种 ASS 样式
   - **导出格式**：SRT / ASS
3. 点击"导出结果"
4. 系统会自动保存修改 → 生成文件 → 标记任务为 done

### 放弃成果

点击"放弃成果"会将任务状态设置为 `error`，表示放弃本次翻译结果。

## 字幕导出与格式

### 输出文件命名

```
原视频文件名.{语言代码}.{格式}

例如：阿索卡 - S01E01 - 第 1 集.zh-CN.srt
```

### SRT 格式

- 纯文本字幕格式，兼容所有播放器
- 支持双语（原文+译文分行显示）
- 不包含任何样式信息

### ASS 格式

- 高级字幕格式，支持样式、定位、特效
- 如果原始字幕是 ASS，会优先保留原始时间轴和事件结构
- 5 种预设样式：
  - **简洁双语**：译文大字 + 原文小字灰色
  - **影院风格**：译文 54px + 原文字幕 24px
  - **学习模式**：译文 46px + 原文 34px
  - **紧凑模式**：译文 40px + 原文 26px
  - **单色模式**：统一颜色，轻阴影

### `__SUBX_FMT_N__` 占位符

这是系统内部使用的格式化保护标记。在以下情况下你可能会看到：

- 核对页的"当前译文"或"最终导出"文本中
- AI 返回的原始响应中

**这些占位符不会出现在最终导出的字幕文件中**。系统在写入文件前会自动清除所有这类标记。

## 日志与排查

### 任务日志

每个任务的详细执行日志保存在两个位置：

| 位置 | 路径 | 保留策略 |
|------|------|---------|
| 数据库 | `task_logs` 表 | 随任务删除而删除 |
| 文件系统 | `db/logs/task_<任务ID>.log` | 受"日志保留天数"控制 |

查看文件日志：

```bash
docker exec subx cat /app/db/logs/task_<任务ID>.log
```

### 任务失败排查步骤

1. **进入任务详情页**：查看错误日志和诊断信息
2. **检查原始响应**：展开"诊断信息"区域，查看 AI 返回的原始内容
3. **检查容器日志**：
   ```bash
   docker compose logs -f subx
   ```
4. **常见失败原因**：

| 错误类型 | 可能原因 | 解决方案 |
|----------|---------|---------|
| 鉴权失败 (401) | API Key 无效或格式错误 | 检查 API Key 是否正确 |
| 速率限制 (429) | 调用过于频繁 | 降低并发数，等待重试 |
| 请求超时 | 网络不通或模型响应慢 | 检查网络，考虑更换模型 |
| 解析异常 | AI 返回格式不规范 | 检查 AI 返回内容，降低 chunkSize |
| 流式请求失败 | 连接中断 | 切换为非流式模式 |

### 校验日志

翻译过程中会输出如下日志：

```
[校验] 块 #1 AI 返回 140/140，有效 140/140，未改动原文: 10 条(将进入核对)
```

- **AI 返回**：AI 实际返回的条目数
- **有效**：被接受的条目数（非空译文）
- **未改动原文**：AI 未翻译的条目（音乐符号、专有名词、繁简同形等），将在核对页中以 `accepted_same` 状态列出

## 常见问题

### 安装与部署

**Q：容器启动后页面无法访问？**

检查端口映射、防火墙、容器是否正常运行：`docker compose ps && docker compose logs subx`

**Q：HTTPS 反向代理后登录异常？**

确保反向代理透传了 `X-Forwarded-Proto: https` 请求头。

**Q：容器内看不到媒体文件？**

- 检查 `docker compose` 中的 `volumes` 挂载是否正确
- 确认填写的是容器内路径（如 `/media/tv`），不是宿主机路径（如 `/volume1/tv`）
- 确认容器用户对目录有读取权限

### 翻译相关

**Q：翻译结果全部是原文，没有译文？**

AI 返回了翻译但被校验拒绝。查看任务日志中的"未改动原文"数。如大量条目为原文，检查：
- API Key 对应的模型是否有余额
- 模型是否支持目标语言翻译
- chunkSize 是否过大导致模型输出截断

**Q：翻译后字幕时间轴偏移？**

SubX 保留原始时间轴，不应出现偏移。如确实偏移，可能是原始字幕文件本身的时间轴问题。

**Q：能否只翻译部分字幕？**

目前不支持选择性翻译部分条目。但可以在翻译完成后进入核对页，对不满意的条目单独重新翻译。

**Q：支持哪些 AI 模型？**

所有兼容 OpenAI 接口的模型都可以使用，包括：
- OpenAI：GPT 系列
- DeepSeek：deepseek-chat / deepseek-v3 等
- Gemini（通过兼容接口）
- 第三方代理/中转服务

### 核对页

**Q：翻译完成但核对页没有内容？**

核对页首次打开时，如果任务没有 `task_review_entries`（如旧任务或 error 任务），系统会自动从原始字幕文件生成基础条目。

**Q：如何修复 AI 翻译"串位"的问题？**

在核对页使用"译文上移"/"译文下移"按钮调整。如果全部偏移了一行，选中偏移起始条目，点"译文下移"。

**Q：核对页的修改会自动保存吗？**

不会自动保存。修改后需点击"保存修改"按钮。但在导出、批量重译前会自动保存。

**Q：为什么核对页能看到 `__SUBX_FMT_1__` 这样的标记？**

这些是系统内部的格式化保护标记（保护 `<i>` 等 HTML 标签在翻译过程中不被 AI 修改）。在最终导出时会自动清除，不会出现在字幕文件中。

### 其他

**Q：如何备份数据？**

备份 `db/` 挂载目录即可，包含：数据库文件、任务日志、AI 响应日志（日志保留天数内）。

**Q：升级后设置会丢失吗？**

不会。设置保存在数据库中，只要 `db/` 目录持续挂载就不会丢失。

**Q：支持繁体中文翻译吗？**

支持。设置"目标语言"为 `zh-TW` 即可。但 AI 模型必须支持繁体中文输出。

**Q：支持从视频中提取双语字幕吗？**

SubX 提取的是视频内嵌的单语言字幕。如需双语，选择"双语对照"输出模式，系统会保留原文并在上方/下方添加译文。

**Q：任务日志文件会持续增长吗？**

不会。日志文件受"日志保留天数"控制，默认 5 天自动清理。同时在数据库中的 `task_logs` 表也会随清空历史操作而被删除。

**Q：如何提高翻译成功率？**

1. 增大分块大小（如 8000-10000），减少上下文断裂
2. 使用翻译能力更强的模型
3. 对于复杂字幕，翻译完成后进入核对页手动调整
4. 添加术语表改善专有名词翻译质量
